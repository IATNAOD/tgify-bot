# 1.2.0

## API Updates

- Support for [API 8.3](https://core.telegram.org/bots/api-changelog#february-12-2025).
- All methods and update types up to this API version are now fully supported.

## You can now send gifts via the bot

```TS
const { Telegraf } = require("@tgify/bot");

const BOT_TOKEN = "xxx:xxx";

const bot = new Telegraf(BOT_TOKEN);

bot.start(async (ctx) => {
	const availableGifts = await bot.getAvailableGifts();
    // OR bot.telegram.getAvailableGifts();

	await ctx.sendGift(availableGifts.gifts[0].id, { text: "*its* for you", text_parse_mode: "MarkdownV2" });
    // OR ctx.telegram.sendGift(availableGifts.gifts[0].id, { user_id: ctx.from.id, text: "*its* for you", text_parse_mode: "MarkdownV2" });
});

bot.launch();

// Enable graceful stop
process.once("SIGINT", () => bot.stop("SIGINT"));
process.once("SIGTERM", () => bot.stop("SIGTERM"));

```

## Support for the following methods has also been added:

- `setUserEmojiStatus('userId',{ emoji_status_custom_emoji_id: '', emoji_status_expiration_date: '' })` - changes the emoji status for a given user that previously allowed the bot to manage their emoji status via the Mini App method requestEmojiStatusAccess

- `savePreparedInlineMessage('userId', result: 'tg.InlineQueryResult', { allow_user_chats: true, allow_bot_chats: true, allow_group_chats: true, allow_channel_chats: true })` - stores a message that can be sent by a user of a Mini App

- `editUserStarSubscription('userId', 'paymentChargeId', true)` - allows the bot to cancel or re-enable extension of a subscription paid in Telegram Stars

- `verifyUser`, `verifyChat`, `removeUserVerification`, `removeChatVerification` - they are responsible for third-party verification. You can find more details [here](https://telegram.org/verify#third-party-verification).
